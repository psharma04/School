Inferior Security Algorithm - Documentation


What is the ISC?

The ISC is a simple hashing algorithm by @MaXiTOR994 and @RBXII3. It was designed to be lightweight, have less than 250 lines of code and stand up to a basic bruteforce script, as well as a reverse-engineered decoder.
The ISC accomplished all of these requirements. Before minifying, the file was only 143 lines long, and took up a mere 7KB. It ran smoothly on everything from a MacBook Pro to a Raspberry Pi, and the bruteforce test script took over a month to crack a 4 digit code.

How does it work?

The ISC is entirely made in Python 2, and is heavily based on prime numbers and pseudo-random number generators.

NOTE: Throughout this page, we will work through the code responses as the user sees them. Therefore, we need to have sample inputs as well.

  • Unencrypted data: Hello123
  • Password: n4men0tfound
  • Seed: CSEC4472

There are 4 steps:

First, the user inputs some data to encrypt. They are then prompted for a password and seed. Note how the program does not display the password, but still captures the input.



Then, the script uses this seed to generate a large number.



After that, the password and unencrypted data are translated into numbers, and the seed, password, data and 11 prime numbers are all multiplied together.



Finally, the hash is outputted for the user, the script opens up this website, and automatically exits.




This algorithm is extremely simple. However, it is not very efficient. MD5, a well known, widely used hashing algorithm, when coded in Python, is only 6.11KB. Another algorithm, SHA-512, is even smaller, at only 4.52KB. This shows the difference between 2 months and 10 years of algorithm development. SHA-512 is much more secure than ISC, and even the now-outdated MD5 is more secure and more efficient.

That's not to say it can't be improved, though. You can check out the source code at https://github.com/psharma04/SDL-Cryptography-Algorithm. Who knows, maybe an implementation of ISC might someday become as popular as MD5 and SHA!
